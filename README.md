# ІДС "Глосарій-КБ"

Сучасна інформаційно-довідкова система для каталогізації, інтелектуальної обробки та пошуку військових термінів зі зв'язку, кібербезпеки та РЕБ. 
Розроблена для автономної роботи в закритій мережі.

## 🛠 Стек технологій

- **Frontend**: React + Vite + Tailwind CSS
- **Backend**: Node.js + Express
- **База даних**: MySQL
- **AI**: Локальна LLM через Ollama (`llama3` + `nomic-embed-text`)
- **Деплой**: Docker + Docker Compose

## 👥 Ролі та доступ

Система використовує сувору рольову модель (RBAC) та маркування документів.

**Рівні ролей:**
- `admin`: Повний доступ (керування юзерами, термінами, документами, статистикою).
- `operator`: Може завантажувати документи, редагувати та підтверджувати терміни від ШІ.
- `user`: Тільки перегляд, базовий та AI-пошук.

**Грифи секретності (Clearance):**
- `Public` (Відкрито): бачать абсолютно всі користувачі.
- `DSP` (ДСК): бачать користувачі з рівнями доступу `operator` та `admin` (якщо їм надано цей гриф).
- `Secret` (Таємно): бачать виключно користувачі з грифом `Secret` (зазвичай `admin`).

> **Важливо:** Автентифікація реалізована через JWT-токени (`access` - 15хв, `refresh` у httpOnly cookie - 7 днів).

## ⚙️ Змінні середовища (.env)

Для роботи бекенду необхідно створити файл `.env` або передати змінні через Docker:

```env
DB_HOST=db                 # Хост бази даних
DB_USER=root               # Користувач БД
DB_PASSWORD=qwerty123      # Пароль до БД
DB_NAME=guid_book          # Назва БД
PORT=3001                  # Порт бекенду
JWT_SECRET=secret_key      # Секрет для Access токенів
JWT_REFRESH_SECRET=ref_key # Секрет для Refresh токенів
OLLAMA_URL=http://host.docker.internal:11434 # URL до Ollama
NODE_ENV=production        # Середовище
```

## 🚀 Запуск

Проєкт повністю контейнеризований і готовий до запуску однією командою:

```bash
docker-compose up -d --build
```
