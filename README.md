# 🛡️ Інформаційно-довідкова система "Глосарій-КБ"

Сучасна аналітична платформа для каталогізації, інтелектуальної обробки та пошуку військових термінів зі зв'язку, кібербезпеки та РЕБ. Розроблена для автономної роботи в закритих мережах з використанням локального штучного інтелекту.

---

## ✨ Основні можливості

- 🧠 **ШІ-Аналіз документів:** Автоматичне видобування термінів з PDF, DOCX та XLSX файлів за допомогою локальної LLM.
- 🔍 **Розумний пошук:** Семантичний (векторний) пошук за змістом та класичний повнотекстовий пошук.
- 🔐 **Безпека та RBAC:** Строга рольова модель та грифування доступу до інформації. Всі дії логуються, JWT-токени надійно захищені (httpOnly cookies).
- 📊 **Аналітика:** Дашборд зі статистикою наповнення бази та контролем актуальності термінів.
- ⭐ **Персоналізація:** Збереження історії пошукових запитів та обраних термінів для кожного користувача.
- 🌙 **Темна тема:** Вбудований Dark Mode для комфортної роботи під час нічних чергувань.
- 📄 **Експорт:** Вивантаження готових довідників у форматах PDF та CSV.

---

## 🛠 Технологічний стек

- **Frontend**: React 19, Vite, Tailwind CSS
- **Backend**: Node.js, Express, JWT, Server-Sent Events (SSE)
- **База даних**: MySQL 8.0
- **Штучний інтелект**: Ollama (моделі `llama3` для тексту та `nomic-embed-text` для векторів)
- **Інфраструктура**: Docker, Docker Compose, Nginx

## 👥 Ролі та Грифи секретності

Система реалізує багаторівневий доступ до інформації.

### Ролі користувачів:
- **`admin` (Адміністратор):** Повний контроль над системою. Керування користувачами (створення, блокування), видалення документів, редагування бази термінів.
- **`operator` (Оператор):** Право завантажувати нові документи, запускати ШІ-аналіз та верифікувати видобуті терміни перед додаванням їх до бази.
- **`user` (Користувач):** Базовий доступ для перегляду довідника, використання класичного та семантичного пошуку.

### Грифи доступу (Clearance):
Кожен документ (і всі терміни в ньому) маркується грифом обмеження. Користувач бачить лише ті терміни, рівень доступу до яких йому дозволено:
- 🟢 **Public (Відкрито):** Доступно всім користувачам системи.
- 🟡 **DSP (ДСК - Для службового користування):** Доступно Операторам та Адміністраторам.
- 🔴 **Secret (Таємно):** Доступно виключно Адміністраторам (або користувачам, яким явно надано цей гриф).

## ⚙️ Вимоги до середовища (Prerequisites)

1. Встановлений **Docker** та **Docker Compose**.
2. Встановлений локальний сервер **Ollama**.
3. Завантажені моделі ШІ в Ollama:
   ```bash
   ollama pull llama3
   ollama pull nomic-embed-text
   ```

## 🚀 Розгортання та Запуск

### 1. Налаштування змінних середовища
Система підтримує конфігурацію через файл `.env` у теці `backend/`, проте базові налаштування вже інтегровані в `docker-compose.yml`.

Основні змінні:
```env
DB_HOST=db
DB_USER=root
DB_PASSWORD=qwerty123       # Обов'язково змініть у production!
DB_NAME=guid_book
PORT=3001
JWT_SECRET=super_secret_key # Згенеруйте надійний ключ
JWT_REFRESH_SECRET=super_refresh_key
OLLAMA_URL=http://host.docker.internal:11434
NODE_ENV=production
```

### 2. Запуск контейнерів
Відкрийте кореневу теку проєкту в терміналі та виконайте команду:

```bash
docker-compose up -d --build
```

Система автоматично:
1. Підніме базу даних MySQL.
2. Створить усі необхідні таблиці (міграції).
3. Збере фронтенд і запустить Nginx.
4. Підніме Node.js бекенд.

Додаток буде доступний за адресою: **http://localhost**
